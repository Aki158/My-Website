## 📝説明
このサイトは、私のレジュメとポートフォリオを紹介した静的なWebサイトです。

このサイトでは、採用に関わる企業の方やソフトウェアエンジニアに見てもらうことを想定して作成しています。

ポートフォリオのページは、注目してもらいたい成果物から順に展示しています。

基本的な機能として、レジュメとポートフォリオの表示/個々の成果物に関係するサイトへのアクセスができます。

ユーザーは、レジュメとポートフォリオから下記のような情報を確認することができます。

### 補足
[説明](#説明)で登場する用語について補足します。

用語の意味がわからない時は、下記表を確認してください。

| 用語 | 意味 |
| ------- | ------- |
| レジュメ | 職業的経験、スキル、教育、およびその他の関連する成果や資格をまとめた文書のことです。 |
| ポートフォリオ | 個人の技能、経験、成果を示すために使用される作品集のことです。 |

## 🚀使用方法
### レジュメページ
1. [URL](#URL)にアクセスする
2. レジュメページを確認する

### ポートフォリオページ
1. [URL](#URL)にアクセスする
2. レジュメページの個人開発のPortfolioをクリックする
3. ポートフォリオの目次を確認し、見たい成果物をクリックする
4. 成果物の情報を確認する
5. 目次に戻りたい場合は、右下に表示されている ⬆︎Back to Top をクリックする

## 🙋使用例
一通りの手順のイメージは[デモ](#デモ)を参考にしてください。

### レジュメページ
使用方法の通りなので、使用例は省略します。

### ポートフォリオページ
1. [URL](#URL)にアクセスする
2. レジュメページの個人開発のPortfolioをクリックする
3. ポートフォリオの目次を確認し、`Markdown-to-HTML`を見たいのでクリックする
4. `Markdown-to-HTML`について説明を読む
5. もう一度目次を見たいので ⬆︎Back to Top をクリックする

【使えるかもしれない素材】

静的ウェブアプリケーションの開発および本番環境へのリリースを理解するためのプロジェクトです。

プロジェクトを実施する前提条件としてまずは、自身のウェブサーバをセットアップする方法を学びました。
- 仮想マシン
- Ubuntu
- Amazon EC2

次に、ウェブサーバの役割と HTTP 通信の基本を理解し、NGINX を活用したリクエストのハンドリング方法について学びました。

最後に、インターネットの基盤である DNS について理解し、TLS によるセキュリティ確保の重要性を学びました。

これらの知識を活用して自身の静的なウェブサイトをデプロイしました。

このプロジェクトでは、職務経歴書とポートフォリオ・プロジェクトを紹介する静的ウェブサイトを作成しました。

ユーザーは、職務経歴書とこれまでに作成したポートフォリオ・プロジェクトの一覧を見ることができます。

このプロジェクトは、一人ですべての開発を行いました。

このプロジェクトでは、HTML/CSS/JavaScriptで静的なウェブサイトを作成し自身のサーバーにデプロイする経験が得られました。
